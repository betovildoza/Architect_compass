Guía de Profesionalización: Compass CLI

Transformación del motor Architect's Compass de script pasivo a herramienta de comandos profesional.

Reemplazar JSON por Argparse (o Typer)

Actualmente, el script es "pasivo": se ejecuta y lee lo que encuentra en su propia carpeta. Para que sea una CLI real, debe aceptar parámetros que alteren su comportamiento sin editar el código.

¿Qué ganamos? Flexibilidad total para automatizar auditorías.
  • compass --config mi_config.json: Permite usar reglas personalizadas para un proyecto específico.
  • compass --path ./otro_proyecto: Analiza cualquier carpeta del disco desde cualquier ubicación.
  • compass --export: Define si queremos la salida completa o solo un archivo específico (ej: solo el Atlas JSON).
import argparse
parser = argparse.ArgumentParser()
parser.add_argument("--path", default=".")
parser.add_argument("--config", help="Ruta al JSON")
args = parser.parse_args()

Definir Puntos de Entrada (Entry Points)

Si querés publicar esto en PyPI (para instalar con pip install architect-compass), necesitás que el sistema reconozca el comando de forma nativa.

Configurando un archivo pyproject.toml o setup.py, le decís al sistema operativo: "Cuando el usuario escriba compass, ejecutá la función main()".

Esto elimina para siempre la necesidad de andar lidiando con archivos .bat o variables de entorno PATH manualmente. 
[project.scripts]
compass = "architect_compass:main"

Manejo de Rutas Relativas y Absolutas

Una CLI profesional no puede asumir que su configuración está "al lado". El binario vive en la carpeta de Python, pero el usuario está en su carpeta de proyecto.

La búsqueda del mapper_config.json debe tener esta prioridad (Jerarquía):

  1. Argumento: La ruta explícita pasada con --config.
  2. Raíz del Proyecto: Si el proyecto que se analiza tiene su propio JSON, tiene prioridad.
  3. Global: Una carpeta fija en el usuario (ej: ~/.config/compass/) para preferencias generales.

Salida por Consola Atractiva (Rich)

Las CLI modernas deben ser visuales. Usar la librería Rich permite que la terminal deje de ser una pared de texto aburrida.

  • Progress Bars: Visualizar el escaneo cuando hay miles de archivos.
  • Tablas: Mostrar el resumen de salud estructural y archivos huérfanos directamente en pantalla sin abrir archivos externos.
  • Colores: Semántica visual inmediata (Rojo para errores críticos, Verde para salud óptima).
from rich.console import Console
console = Console()
console.print("[bold green]Salud óptima detectada[/bold green] :compass:")

Estructura de Comandos (Sub-commands)

Organizar la lógica en acciones específicas mejora la experiencia de usuario. En lugar de una ejecución única, dividimos las tareas:

  • compass scan: Ejecuta el análisis y auditoría completa.
  • compass init: Crea un archivo mapper_config.json por defecto en la carpeta actual (para que el usuario empiece rápido).
  • compass graph: Se enfoca solo en procesar y abrir la visualización del grafo .dot.
